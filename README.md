# Reflexión – Proyecto No.3: Portafolio

## Objetivo del proyecto

La idea original del proyecto era diseñar un portafolio con una estética minimalista y profesional. El objetivo principal era crear una página capaz de captar la atención de empresas formalizadas interesadas en incorporar desarrolladores frontend a sus equipos de trabajo.

Respecto al diseño visual, considero que el objetivo sí se logró; sin embargo, siento que el contenido fue el elemento que rompió parcialmente esa estética. Al momento de seleccionar los proyectos para incluir dentro del portafolio, me percaté de que todavía no cuento con trabajos de gran escala. Dos de los tres proyectos agregados son relativamente pequeños, lo cual limita un poco la percepción de mis habilidades como desarrollador.

El proyecto que considero más alineado con una estética empresarial formal fue el desarrollado en Software 1, ya que estuvo basado en las necesidades y preferencias de una empresa real.

---

## Tecnologías utilizadas

Las tecnologías utilizadas en este proyecto fueron:

- JavaScript
- Vite
- React
- GSAP

Las primeras tres tecnologías fueron seleccionadas porque representan algunas de las principales herramientas aprendidas durante el curso de Sistemas y Tecnologías Web. La experiencia acumulada durante el curso me permitió trabajar con mayor comodidad y obtener un mejor resultado final.

Respecto a GSAP, decidí implementarlo para agregar un elemento diferenciador al portafolio mediante animaciones y movimiento dentro de la página. La experiencia utilizando esta librería fue bastante positiva: cuenta con una instalación rápida, buena documentación y alternativas interesantes a herramientas como `useEffect` de React, las cuales considero más cómodas para ciertos escenarios de animación.

Para este proyecto únicamente exploré el nivel introductorio de GSAP, pero incluso con ese conocimiento básico quedé satisfecho con el resultado y considero seguir utilizándolo con mayor profundidad en proyectos futuros.

---

## Tecnologías no utilizadas

Los lenguajes que decidí no utilizar fueron TypeScript y Go.

En el caso de TypeScript, no lo utilicé porque todavía no me siento completamente cómodo trabajando con él. Sé que es una versión más robusta de JavaScript gracias al sistema de tipado, pero actualmente estoy más acostumbrado a la simplicidad y flexibilidad de JavaScript. Aun así, considero importante explorar TypeScript con mayor profundidad en futuros proyectos.

Respecto a Go, decidí no implementarlo porque no vi necesidad de desarrollar un backend para este portafolio. Desde el inicio tenía planeado subir los videos mediante una herramienta externa para reducir el tiempo de carga de la página y poder priorizar otros aspectos importantes del proyecto, como las animaciones y la experiencia visual.

En cuanto a prácticas de desarrollo, no implementé pruebas automatizadas debido a la ausencia de un backend y porque sentí que resultaban algo redundantes para este proyecto específico. Storybook sí hubiera sido una herramienta útil para probar la distribución de imágenes y videos dentro de la interfaz, pero olvidé integrarlo desde la estructura inicial del proyecto.

---

## Nivel de riesgo y aprendizaje

Definitivamente decidí jugar relativamente seguro con JavaScript y Vite. Llevo aproximadamente dos meses practicando y experimentando con estas tecnologías, por lo que representaban una apuesta confiable para el desarrollo del proyecto.

En un nivel de riesgo intermedio colocaría React. Aunque también tengo tiempo utilizándolo, todavía no me siento completamente cómodo con algunos conceptos importantes, especialmente relacionados con `useState`, `useEffect` y ciertas limitaciones o patrones propios de React que no tuvimos oportunidad de explorar completamente en clase.

La apuesta más arriesgada fue GSAP. Descubrí esta librería hace menos de dos semanas gracias a un amigo que se interesó en ella mientras investigaba la página oficial de GTA 6. Al momento de implementarla en el proyecto, únicamente tenía como experiencia aproximadamente seis videos de YouTube sobre la librería. Aun así, utilizando la documentación oficial, ejemplos prácticos y manteniendo un enfoque relativamente simple, considero que se logró un resultado bastante decente.

---

## Mejoras futuras

Si tuviera una semana adicional para trabajar en el proyecto, me gustaría:

- Explorar animaciones más avanzadas para los elementos dinámicos de la página.
- Implementar botones para visualizar proyectos desplegados mediante GitHub Pages.
- Incrementar la cantidad de proyectos incluidos en el portafolio.
- Utilizar una mejor plataforma externa para alojar los videos.
- Mejorar el tiempo de respuesta y rendimiento general de la página.
- Agregar una sección de experiencia con logos de tecnologías y lenguajes que manejo de forma competente.
